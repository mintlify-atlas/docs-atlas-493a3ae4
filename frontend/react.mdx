React is a **JavaScript library** used to create **interactive** user interfaces. Created by Facebook/Meta, it's the industry standard for frontend development.

## What is React?

React shines in applications with many possible user interactions — like Instagram, YouTube, Netflix, Spotify, and LinkedIn, all of which use React.

## Single Page Application (SPA)

<Tabs>
  <Tab title="Concept">
    React generates a **SPA** — the application's HTML is practically empty. Content is dynamically inserted via JavaScript.

    Instead of having one HTML file per page, React manages all pages via JavaScript with just one base HTML.
  </Tab>
  
  <Tab title="Example">
    ```html
    <!-- index.html (empty) -->
    <div id="root"></div>

    /* React injects content here */
    <div id="root">
      <h1>Hello World</h1>
    </div>
    ```
  </Tab>
</Tabs>

<Warning>
If you disable JavaScript in the browser, you won't see anything in a React SPA.
</Warning>

## Components

In React, the application is divided into **components** — think of them as LEGO pieces that combine to form complete pages.

Each component is a **JavaScript function** that returns **JSX**. By convention, use **PascalCase** to name them (e.g., `AddTask`, `TaskList`).

```jsx src/components/MyComponent.jsx
function MyComponent() {
  return (
    <div>
      <h1>Hello World</h1>
    </div>
  );
}

export default MyComponent;
```

<Note>
Component files use the `.jsx` extension, created inside the `src/components/` folder.
</Note>

<Warning>
The `return` of a component can only have **one root element**. Use a `<div>` or a Fragment (`<></>`) to wrap multiple elements.
</Warning>

## JSX

JSX is a special syntax: **HTML mixed with JavaScript**.

<Tabs>
  <Tab title="JavaScript in JSX">
    To use JavaScript inside JSX, wrap it with `{}` (curly braces).

    ```jsx
    const name = "Felipe";

    return (
      <div className="container">
        <h1>Hello, {name}</h1>
        <button onClick={() => alert('clicked!')}>
          Click here
        </button>
      </div>
    );
    ```
  </Tab>
  
  <Tab title="Key Differences">
    - Use `className` instead of `class` for CSS classes
    - Events like `onClick` and `onChange` receive functions inside curly braces
    - All JSX tags must be closed (even `<img />`, `<input />`)
  </Tab>
</Tabs>

## State with `useState`

State is a special React variable: **when you change the state, the interface automatically updates** (re-render).

It's created with the `useState` Hook, imported from React. Returns an array with `[value, update function]`.

```jsx
import { useState } from 'react';

function App() {
  const [message, setMessage] = useState('Hello World');

  return (
    <div>
      <h1>{message}</h1>
      <button onClick={() => setMessage('Clicked!')}>
        Change
      </button>
    </div>
  );
}
```

<Tip>
Use state whenever you want to **react to user interactions** — that's where the name "React" comes from.
</Tip>

## Props

Props are the way to **pass data from a parent component to a child component**.

<CodeGroup>
```jsx Parent Component (App)
<Tasks tasks={taskList} />
```

```jsx Child Component (Tasks)
function Tasks(props) {
  return (
    <ul>
      {props.tasks.map(task => (
        <li>{task.title}</li>
      ))}
    </ul>
  );
}
```
</CodeGroup>

<Info>
Everything you pass as an attribute becomes available inside `props.attributeName`.
</Info>

## Side Effects with `useEffect`

The `useEffect` executes a function whenever a value in the dependency list (second parameter) changes.

<Tabs>
  <Tab title="With Dependencies">
    ```jsx
    // Runs when 'tasks' changes
    useEffect(() => {
      localStorage.setItem(
        'tasks',
        JSON.stringify(tasks)
      );
    }, [tasks]);
    ```
  </Tab>
  
  <Tab title="Empty Dependencies">
    Empty dependency list `[]`: function executes **only once**, when component is mounted for the first time.

    ```jsx
    // Runs only once (on mount)
    useEffect(() => {
      async function fetchTasks() {
        const res = await fetch(url);
        const data = await res.json();
        setTasks(data);
      }
      fetchTasks();
    }, []);
    ```
  </Tab>
</Tabs>

<Warning>
Don't use `async` directly in the `useEffect` function. Create an async function inside and call it.
</Warning>

## localStorage Persistence

The `localStorage` persists data in the browser even after closing the tab or browser.

```jsx
// Initialize state with data from storage
const [tasks, setTasks] = useState(
  JSON.parse(localStorage.getItem('tasks')) || []
);

// Save to storage when tasks change
useEffect(() => {
  localStorage.setItem('tasks', JSON.stringify(tasks));
}, [tasks]);
```

<Note>
Stores only **strings**, so use `JSON.stringify()` to save and `JSON.parse()` to retrieve.
</Note>

## Routing with React Router Dom

React Router manages routes (pages) in an SPA using JavaScript, without multiple HTMLs.

**Install:** `npm install react-router-dom@6.26.1`

Create a `src/pages/` folder for page components. Pages are common components — the name is just semantic.

| Hook / Component | Use |
|------------------|-----|
| `createBrowserRouter` | Creates the router with the list of routes (path + element). |
| `RouterProvider` | Component that wraps the application and injects the router. |
| `useNavigate()` | Hook to navigate between routes. `navigate('/route')` or `navigate(-1)` to go back. |
| `useSearchParams()` | Hook to read query params from URL. E.g., `?title=foo&desc=bar`. |

<Warning>
When passing functions to `onClick`, never call them directly (`onClick={func()}`). Pass the reference (`onClick={func}`) or use an arrow function (`onClick={() => func(param)}`).
</Warning>

## Consuming APIs with `fetch`

The most common flow for APIs in React:

<Steps>
  <Step title="Component is mounted">
    Component renders for the first time
  </Step>
  <Step title="useEffect calls the API">
    Effect with empty dependencies `[]` triggers
  </Step>
  <Step title="Data saved in state">
    `setState` updates the component state
  </Step>
  <Step title="Interface updated">
    Component re-renders with new data
  </Step>
</Steps>

```jsx
useEffect(() => {
  async function fetchTasks() {
    const response = await fetch(
      'https://jsonplaceholder.typicode.com/todos?_limit=10',
      { method: 'GET' }
    );
    const data = await response.json();
    setTasks(data);
  }
  fetchTasks();
}, []);
```
