## Quick Reference

### Basic SELECT

```sql Query all columns
SELECT * FROM users;
```

```sql Query specific columns
SELECT name, email FROM users;
```

```sql Filter results
SELECT * FROM users WHERE age > 18;
```

```sql Sort results
SELECT * FROM users ORDER BY name ASC;
```

### Data Manipulation

```sql Insert data
INSERT INTO users (name, email, age) 
VALUES ('John Doe', 'john@example.com', 25);
```

```sql Update data
UPDATE users 
SET email = 'newemail@example.com' 
WHERE id = 1;
```

```sql Delete data
DELETE FROM users WHERE id = 1;
```

### Table Operations

```sql Create table
CREATE TABLE users (
  id SERIAL PRIMARY KEY,
  name VARCHAR(100) NOT NULL,
  email VARCHAR(255) UNIQUE NOT NULL,
  age INTEGER,
  created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP
);
```

```sql Alter table
ALTER TABLE users ADD COLUMN phone VARCHAR(20);
```

```sql Drop table
DROP TABLE IF EXISTS users;
```

## Common Patterns

<Accordion title="Pagination">
```sql Paginate results
SELECT * FROM users 
ORDER BY id 
LIMIT 10 OFFSET 20;
```

This retrieves 10 users starting from the 21st record (useful for pagination).
</Accordion>

<Accordion title="Count Records">
```sql Count all records
SELECT COUNT(*) FROM users;
```

```sql Count with condition
SELECT COUNT(*) FROM users WHERE age > 18;
```

```sql Group by and count
SELECT country, COUNT(*) as user_count 
FROM users 
GROUP BY country;
```
</Accordion>

<Accordion title="Search Patterns">
```sql Exact match
SELECT * FROM users WHERE name = 'John Doe';
```

```sql Pattern matching
SELECT * FROM users WHERE name LIKE 'John%';
```

```sql Case-insensitive search (PostgreSQL)
SELECT * FROM users WHERE name ILIKE '%john%';
```
</Accordion>

## Database-Specific Dialects

<Note>
SQL syntax can vary slightly between database systems (PostgreSQL, MySQL, SQLite, SQL Server). Always check your specific database documentation for exact syntax.
</Note>

<Tabs>
  <Tab title="PostgreSQL">
    PostgreSQL is a powerful, open-source object-relational database system.
    
    ```sql PostgreSQL-specific features
    -- Serial auto-increment
    CREATE TABLE users (
      id SERIAL PRIMARY KEY
    );
    
    -- JSON support
    SELECT data->>'name' FROM users;
    
    -- Array types
    SELECT * FROM posts WHERE 'sql' = ANY(tags);
    ```
  </Tab>
  
  <Tab title="MySQL">
    MySQL is one of the most popular open-source databases.
    
    ```sql MySQL-specific features
    -- Auto-increment
    CREATE TABLE users (
      id INT AUTO_INCREMENT PRIMARY KEY
    );
    
    -- Limit syntax
    SELECT * FROM users LIMIT 10;
    ```
  </Tab>
  
  <Tab title="SQLite">
    SQLite is a lightweight, file-based database.
    
    ```sql SQLite-specific features
    -- Auto-increment
    CREATE TABLE users (
      id INTEGER PRIMARY KEY AUTOINCREMENT
    );
    
    -- Simpler syntax, fewer data types
    ```
  </Tab>
</Tabs>

## Security

<Warning>
**Never** concatenate user input directly into SQL queries. This opens your application to SQL injection attacks.
</Warning>

<CodeGroup>
```javascript Bad - SQL Injection Vulnerable
// DON'T DO THIS!
const query = `SELECT * FROM users WHERE email = '${userInput}'`;
```

```javascript Good - Parameterized Query
// DO THIS INSTEAD
const query = 'SELECT * FROM users WHERE email = $1';
const result = await db.query(query, [userInput]);
```
</CodeGroup>
